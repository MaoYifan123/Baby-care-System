# 婴幼儿智能监护系统 - 云边端协同软件

## 项目概述

本项目是**婴幼儿智能监护云边端协同系统**，为武汉大学计算机学院2025-2026学年第二学期《物联网技术与应用》课程设计。系统模拟了真实物联网环境下的云边端协同架构，实现了对婴幼儿体温、心率、哭声、体位等生理指标的实时监测与智能分析。

## 功能特性

| 功能 | 说明 |
|------|------|
| **云边端计算划分** | 边缘层处理实时告警，雾层处理数据汇聚，云层处理趋势分析和智能建议 |
| **流处理引擎** | 边缘节点和云端均配备流处理引擎，基于滑动窗口的实时分析 |
| **一般处理（批处理）** | 支持批量数据导入和历史数据的离线分析 |
| **大数据分析** | 使用分布式计算思想（MapReduce风格）进行多维度数据聚合分析 |
| **智能建议** | 基于规则引擎和大模型风格的知识库，生成个性化护理建议 |
| **实时监控界面** | PyQt5图形界面，实时仪表盘展示数据曲线和告警信息 |

## 计算任务划分

| 计算层 | 任务 | 特点 |
|--------|------|------|
| **边缘层 Edge** | 实时滤波、紧急告警、本地缓存、流统计 | 超低延迟、本地响应 |
| **雾层 Fog** | 数据聚合、短期趋势分析、噪声抑制 | 中等延迟、多节点协调 |
| **云层 Cloud** | 长期趋势分析、异常检测、智能建议、报告生成 | 高计算能力、全局视图 |

## 安装与运行

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 快速启动（完整系统）

```bash
python main.py
```

启动后会自动：
- 启动云端服务（端口 5000）
- 启动边缘计算模拟节点
- 打开 PyQt5 监控仪表盘

### 3. 独立模式

```bash
# 仅云端服务
python main.py --mode cloud --port 5000

# 仅边缘节点
python main.py --mode edge

# 仅GUI（需云端服务已运行）
python main.py --mode gui

# 测试模式（不启动服务，自检所有模块）
python main.py --test
```

### 4. 单独运行模块

```bash
# 云端服务
python cloud_server.py

# 边缘节点（终端模式）
python edge_node.py

# 大数据分析演示
python bigdata_analysis.py

# GUI独立模式
python gui_monitor.py
```

## API 接口

云端服务启动后可访问以下接口：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/health` | GET | 健康检查 |
| `/api/data/stream` | POST | 接收边缘流数据 |
| `/api/data/batch` | POST | 接收批量数据 |
| `/api/realtime` | GET | 获取实时数据 |
| `/api/history/<sensor>` | GET | 获取历史数据 |
| `/api/alerts` | GET | 获取告警记录 |
| `/api/analytics/suggestion` | GET | 获取智能建议 |
| `/api/analytics/trend` | GET | 获取趋势分析 |
| `/api/compute/partition` | GET | 获取计算划分信息 |
| `/api/stats` | GET | 获取系统统计 |
| `/api/session/start` | POST | 开始监护会话 |

6. **大模型风格建议**：基于知识库的多层次护理建议生成

## 注意事项

1. **首次运行**建议使用 `python main.py --test` 检查所有模块是否正常
2. **GUI界面**需要图形环境支持，服务器环境请使用 `--mode cloud`
3. 数据模拟器会自动生成测试数据，无需连接真实传感器
4. 生产环境部署时需将模拟数据替换为真实传感器驱动
